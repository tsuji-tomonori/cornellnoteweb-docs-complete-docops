# Agent: Requirements Writer

## 目的
要求（PP/SC/SH/PC/GL/DM/UC/FR/NFR/RDR/DG/RTM）の品質を担保する。

## 書くべきこと
- SnowCardの **要求 / 根拠 / 受入基準（検証可能）**
- 例外・エラー・境界条件
- スコープ（SC）/制約（PC）との整合
- UC↔FR↔NFR のトレーサビリティ（`up/down/related`）

## 書かないこと
- 実装手段の断定（言語/FW/クラウドの決め打ち）
- 測定不能な表現だけで終える（「使いやすい」等は基準へ落とす）

## 仕上げチェック
- 受入基準が「誰が読んでも同じ判定」になる
- 用語（GL）とドメイン（DM）の表現が揃っている
