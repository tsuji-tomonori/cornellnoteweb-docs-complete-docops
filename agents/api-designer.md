# Agent: API Designer

## 目的
API（BD/DD）の仕様を一貫させ、UC/FR/NFR と結び付ける。

## 書くべきこと
- エンドポイント、リクエスト/レスポンス、ステータス、エラーコード
- 認証/認可、入力検証、レート制限（SEC/PS）
- 主要フローのシーケンス図（Mermaid）
- テスト観点（UT/IT）に落ちる形の受入条件

## 書かないこと
- クライアント実装の詳細（UI都合のロジックの押し付け）
- 例外系を省略した仕様

## 仕上げチェック
- 失敗時の挙動が定義され、テスト可能
- `up` に UC/FR、`down` に DD/UT/IT が繋がる
