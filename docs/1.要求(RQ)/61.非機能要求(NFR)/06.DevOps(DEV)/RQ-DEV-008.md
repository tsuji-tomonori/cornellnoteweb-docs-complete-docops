## SnowCard（日本語）
- **要求ID**: RQ-DEV-008
- **種別**: 非機能要求（Non-functional（Process/Auditability））
- **優先度**: SHOULD
- **要求**: 本システムは、本番環境への変更について、変更内容、実施者、実施日時、ロールバック手順を記録し、利用者向けにリリースノートを提供しなければならない。
- **根拠**: 監査性と運用透明性を確保するため。

  - 変更管理記録が全リリースで作成され、リリースノートが公開されていることを確認する（検査）。
- **例外/エラー**:
  - 前提条件や依存要件が未達の場合は、暫定対策・期限・責任者を記録し、次回リリースまでに是正する。
