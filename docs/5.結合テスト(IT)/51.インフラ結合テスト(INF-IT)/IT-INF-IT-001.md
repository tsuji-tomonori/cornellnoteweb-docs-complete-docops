## 目的
アプリケーション（Front/Back）を含む結合テストとは別に、**インフラ構成要素間の結合**を確認する。

- 配信（CloudFront / S3 など）の経路・キャッシュ・エラーハンドリング
- 認証・認可（必要な場合）の境界
- 監視（ログ、メトリクス、アラーム）の疎通

## テスト観点（例）
- 配信経路
  - 期待するパスで正しいオリジンに到達する
  - キャッシュ無効化（Invalidation）やバージョニングが意図通り
- 証明書 / HTTPS
  - TLS が有効、強制リダイレクト設定がある場合は意図通り
- ログ
  - アクセスログ/アプリログが所定の格納先に出力される
- 監視
  - 主要メトリクス（4xx/5xx、レイテンシ、エラー率）の取得
  - 重大アラームの通知経路が有効（ダミーで発火確認できる範囲で）

## 成果物
- 実施記録（チェック結果、証跡リンク）
- 課題一覧（優先度、対応方針）
